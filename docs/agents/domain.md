# Domain docs

这个文件说明工程类 skills 在探索本仓库时如何读取领域文档和架构决策。

## 探索代码前先读

- 根目录 `CONTEXT.md`
- 根目录 `docs/adr/`

如果这些文件不存在，继续执行即可，不需要因为缺失而报错，也不需要主动创建。`grill-with-docs` 会在术语或决策真正沉淀时再创建这些文件。

## 当前布局

本仓库使用单上下文布局：

```text
/
├── CONTEXT.md
├── docs/adr/
└── plugins/
```

如果以后不同插件或包形成独立上下文，再考虑改为根目录 `CONTEXT-MAP.md` 指向多个上下文。

## 使用领域词汇

当输出 issue 标题、重构建议、诊断假设或测试名称时，优先使用 `CONTEXT.md` 中已经定义的项目术语。

如果需要的概念还没有出现在 `CONTEXT.md` 中，说明这可能是一个需要后续澄清的领域词汇，不要随意发明同义词。

## 标明 ADR 冲突

如果建议和既有 ADR 冲突，必须明确指出冲突点，而不是静默覆盖既有决策。
